HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Child Elements refer to elements nested inside other elements.
  • Parent Elements refer to elements that house all the child elements.
  • "div" element can be used to create a division in a page.
  • "span" element can be used to create an inline division.
  • "h1", "h2", "h3", etc. elements represent the level of heading a given text block represents.
  • "p" element represents a paragraph or block of text.
  • "ul" element represents unordered lists or bullet points.
  • "ol" element represents ordered lists or numbered points.
  • "li" element represents list items.
  • "img" element contains information about images that are displayed on webpages.
  • "src=" attribute defines the location of the image.
  • "br" element creates a line of empty space, or a line break, between two blocks of content.
  • "a" element creates links to the same webpage or other webpages.
  • "href=" attribute points to the URL for the link.
  • "alt=" attribute contains a text string that describes the appearance and functionality of the image.
  • "footer" element appears at the bottom of the page and usually contains author, copyright, contact, sitemap, and navigation.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • inline CSS refers to style an element directly in the HTML file; applies to a specific HTML tag, using a style attribute with a CSS rule.
  • internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • external CSS style sheet allows developers to keep all their CSS rules in a separate file making design changes easier.
  • when using external style sheet, you must link the file using a "link" element placed in the HTML "head" element
  • A selector defines the element or attributes to which the rules or declarations apply
  • Declarations contain two important components: the CSS property we want to apply and the value of the property ex. property is color and the value for the property is blue
  • You can combine two selectors into a single CSS rule with a comma, which allows for declarations to affect both selectors
  • "text-align:" is one way to center a title or heading on a webpage
  • the "display" property assigned with the "block" value to assign each image to start on a new line
  • "font-size:" will adjust the font size of the element
  • ".card" designates this as a class selector, therefore any HTML element assigned to this class "card" will gain these properties
  • "width:" of the element at "?%" of the available width
  • "margin:" property assigns a "?px" to the top and bottom; "auto" value to the left and right
  • "border:" property assigns a border with a width of "?px" solid and a color value
  • "box-shadow:" CSS property adds shadow effects around an elements frame; "?px" value of X axis and "?px" value of Y axis followed by defining a color value

Git

  • "git status": checks what branch we are currently on.
  • "git checkout -b branch-name": creates a new branch and switches to it.
  • "git add -A": stages our data changes.
  • "git commit -m "your message here": commits the data locally with a descriptive message of changes made.
  • "git pull origin main": pulls the base branch "main" to our feature branch.
  • "git push origin "name of feature branch": this will push up your changes to the remote repository Github.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • "const" keyword allows us to create variables that can't be reassigned a value, "locks down" what that variable will be, and once declared it can never be reassigned.
  • A variable name comes after "const", this name must be unique as it will reference the information stored in the variable in our code.
  • Finally we use an assignment operator "=" to assign the specific data that we want stored in our variable.
  • A String - includes all of the data for the variable in quotation marks.
  • Each line of JavaScript ends with a semicolon.
  • "console.log" method to output a message to the web console by adding an argument.
  • "let" keyword
  • Control flow is the order in which a computer executes code in a script.
  • In JavaScript code runs in cronological order.
  • Conditional statements or loops can be used to control the order in which code is executed, or if it is even executed at all.
  • An "if" statement is a way of introducing decision-making into our code. When using "if" statements, code will only execute "if" the statement meets a condition.
  • Use "else if" when we want to provide more conditions than just if.
  • An Array is a single variable that is used to hold a group of related data.
  • An Index is the location identified with a unique number in the array, this allows to access individual data inside the array.
  • Arrays are zero-indexed and sequential increasing by 1 each time.
  • To access a specific piece of data, we start with the name of the variable that stores the array, then we add the index to the end of the array name in square brackets.
  • A "for" loop is used to perform a task on each piece of data, using a few lines of code.
  • To use "for" loops we need the shape's index and the array's length, this is important because the "for" loop needs to know how many times the code will run.
  • Use "length" to find the length of an array.
  • Use "let" keyword to reassign the value of the variable over time.
  • We use "let" followed by a unique variable name and the assignment operator "=" to assign the specific data that we want stored in our variable.
  • A "for" loop contains the following three statements:
    1. First statement determines a starting point for our loop. ex. "let x = 0"
    2. Second statement is the condition. ex. "x < shapes.length"
    3. Final statement is what allows the array to iterate over each time. ex. "x++"
    4. Lastly, we put the code that we want to be executed for each loop inside curly brackets. ex. "{ console.log(shapes[x]); }
    5. ex. of final code: "for(let x = 0; x < shapes.length; x++) { console.log(shapes[x]); }"
  • A "Function" is a set of instructions that tells the computer how to perform a certain task. They do not automatically execute when the JavaScript file is run you must call a function by name in order to execute it.
